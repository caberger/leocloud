The Canonical Application

Repository: leocloud by Christian Aberger

An example how to build multi-tier applications.

Deployed Components

  • nginx http server and reverse proxy
  • quarkus Microprofile Application Server
  • mysql Database Server

Demo Application

The Demo Table is implemented using typescript via webpack, directly served by nginx and fetching the data with REST from the application server. An example for server-side rendered HTML using Qute can be found here

When this application is deployed on a development root server, you can browse the REST api and download its OpenAPI specification. You can also check the health of the server. The API is served by quarkus through the nginx reverse proxy for all requests starting with /api/. .
